Capítulo 02

Instalando OpenClaw do Zero

OpenClaw é a "central de comando" dos seus agentes de IA. Pense nele como um sistema operacional para agentes — ele faz a ponte entre você, seus modelos de IA e o mundo exterior. Após a instalação, você controla tudo por uma interface visual (TUI) — sem mais precisar do terminal no dia a dia.

O Que é o OpenClaw?

💡
Analogia: Se o agente de IA é um funcionário, o OpenClaw é a empresa. Ele gerencia os contratos (APIs), fornece as ferramentas (e-mail, calendário, internet), controla os acessos e mantém tudo funcionando. Você fala com os agentes pelo Telegram; eles executam tudo via OpenClaw. E você monitora tudo pela interface visual do OpenClaw — o Dashboard.

O OpenClaw é instalado na sua VPS e fica rodando como um serviço permanente. Quando você manda mensagem para o seu agente no Telegram, o OpenClaw recebe, processa com o modelo de IA, executa as ações e responde. Tudo automático. O SSH é usado apenas nesta instalação inicial — depois, você gerencia tudo pelo painel visual.

✅
Boa notícia para leigos: O OpenClaw tem um assistente de configuração interativo chamado openclaw onboard. Ele faz perguntas na tela e você só responde — sem precisar editar arquivos manualmente.

Passo 1 — Instalar o Node.js (obrigatório primeiro)

O OpenClaw é feito em Node.js. Node.js é como o "motor" que faz o OpenClaw funcionar — assim como o Java precisa estar instalado para rodar programas Java.

Usamos o NVM (Node Version Manager) para instalar o Node.js — ele é mais seguro e fácil de atualizar depois.

# Baixar e instalar o NVM (gerenciador de versões do Node.js) curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash # Recarregar o terminal para reconhecer o NVM (obrigatório após instalar) source ~/.bashrc # Instalar o Node.js versão 24 (versão recomendada pelo OpenClaw) nvm install 24 nvm use 24 nvm alias default 24 # Verificar se instalou — deve aparecer "v24.x.x" node --version # Verificar o NPM (gerenciador de pacotes) — deve aparecer "10.x.x" ou superior npm --version
⚠️
Node.js mínimo: O OpenClaw requer Node 24 (recomendado) ou Node 22 LTS (versão 22.16 ou superior). Versões mais antigas não funcionarão.

Passo 2 — Instalar o OpenClaw (oficial)

Com o Node.js instalado, instale o OpenClaw usando o script oficial — a forma mais recomendada:

✅
Método recomendado — Mac/Linux/VPS Ubuntu:
Use o script de instalação oficial. Ele verifica dependências, instala e configura tudo automaticamente.
# Mac, Linux ou VPS Ubuntu — script oficial recomendado curl -fsSL https://openclaw.ai/install.sh | bash
💡
Windows: Para Windows, o OpenClaw recomenda usar o WSL2 (Windows Subsystem for Linux). WSL2 é uma camada que permite rodar Linux dentro do Windows — gratuita, oficial da Microsoft, e faz o OpenClaw funcionar de forma mais estável. Se preferir usar o PowerShell nativo, também funciona (mas WSL2 é mais confiável).
# Windows — PowerShell (nativo) iwr -useb https://openclaw.ai/install.ps1 | iex
# Alternativa: instalação via npm (também funciona em qualquer SO) npm install -g openclaw@latest
# Verificar se instalou — deve aparecer a versão atual openclaw --version

Passo 3 — Executar o Assistente de Configuração (onboard)

Agora vem a parte mais fácil: o OpenClaw tem um assistente interativo que guia você por toda a configuração. Execute:

# Iniciar o assistente de configuração e instalar como serviço permanente # "--install-daemon" faz o OpenClaw iniciar automaticamente toda vez que a VPS ligar openclaw onboard --install-daemon

O assistente vai perguntar:

  • 🤖 Qual modelo de IA você quer usar (vamos configurar OpenRouter no próximo capítulo)
  • 📱 Qual canal de mensagens (Telegram, WhatsApp, Discord)
  • 🔑 Suas chaves de API
  • 📁 Onde ficará o workspace dos seus agentes
💡
Dica: Se travar em alguma pergunta, aperte Ctrl+C para sair e execute openclaw onboard novamente. O assistente é tolerante a erros — você pode recomeçar quantas vezes quiser.

Passo 4 — Abrindo o TUI (sua interface de chat) 🎉

Após o onboard, o OpenClaw está rodando. O TUI (Terminal User Interface) é a interface de chat onde você conversa com seu agente — direto no terminal SSH, como uma conversa por texto. Você continua no mesmo terminal onde instalou tudo.

# Se fechou o terminal, conecte novamente via SSH: ssh root@SEU_IP_DA_VPS # Para iniciar o TUI e conversar com o agente: openclaw tui
✅
O que você verá: Uma tela de chat em texto — fundo escuro, mensagens do agente em destaque, e uma barra de status na parte inferior mostrando: connected | idle | agent main | modelo | tokens usados. Você digita sua mensagem e pressiona Enter. O agente responde.

💡
Windows (PowerShell) ou Mac/Linux (Terminal): O TUI funciona igual em qualquer sistema. O que muda é só o programa que você usa para conectar via SSH — no Windows use o PowerShell ou o Windows Terminal; no Mac/Linux use o Terminal nativo.

Como Verificar se Está Tudo Funcionando

# Verificar o status do gateway (serviço principal do OpenClaw) openclaw gateway status # Se aparecer "running" — está funcionando ✅ # Se aparecer "stopped" — iniciar manualmente: openclaw gateway start

Como Usar o TUI no Dia a Dia

O TUI é o seu canal principal com o agente. Você conecta via SSH e conversa normalmente — em português, como se fosse um WhatsApp em modo texto. Tudo que você pedir, o agente executa.

O que fazerComo fazer no TUI
Conversar com o agenteAbra o SSH e digite normalmente
Configurar o agenteDiga ao agente o que mudar — ele atualiza os arquivos
Criar automações"Crie um cron job que faz X todo dia às 8h"
Verificar statusopenclaw gateway status no terminal
Reiniciaropenclaw gateway restart no terminal
Integrar com TelegramPeça ao agente no TUI: "Configure a integração com Telegram"
💡
Dica: Após integrar o Telegram (próximos capítulos), você não precisará mais abrir o SSH para conversar com o agente. Poderá mandar mensagem direto pelo Telegram do celular. O TUI via SSH é mais usado para configurações iniciais e manutenção.

openclaw pluginsGerencia plugins e extensões
